npm install

Quickstart

From clone to dashboard

  1. Install dependencies. Run npm install in the repository.
  2. Create local env. Copy .env.example to .env and keep real secrets local.
  3. Start Crossposter. Run npm run dev:local and open http://localhost:2004.
  4. Add profiles. Use Settings, then Socials, to configure each provider and active profile.
  5. Publish or schedule. Use the Dashboard composer to send posts immediately or queue them locally.

Configuration

Where Crossposter stores settings

Local config

The Settings UI writes provider profiles, active profiles, drafts, scheduled posts, and publish history to poster.config.local.json. The file is gitignored and should stay private.

Environment fallback

Crossposter can also read environment variables, but local config is the normal path because it supports multiple profiles per provider.

Local runtime data

Uploaded media and provider sessions live in local folders such as .poster-uploads, .instagram-sessions, and .pinterest-sessions.

Provider setup

What each profile needs

X / Twitter

Install bird, sign in to X in a supported browser, then configure the bird command and optional browser profile.

LinkedIn

Create a LinkedIn developer app, add the local callback, then connect with w_member_social.

Bluesky and Mastodon

Use a Bluesky app password, or a Mastodon instance URL and access token.

Instagram and Pinterest

Install the Python dependencies. Use one session file or folder per account, and complete platform challenges manually when required.

YouTube and Hacker News

YouTube reads signed-in Chrome cookies or a saved cookie header. Hacker News can use a saved cookie or username/password login.

Dev.to, Nostr, Dribbble, Peerlist

Dev.to uses an API key. Nostr signs with a private key. Dribbble uses OAuth. Peerlist reads Chrome cookies.

Security

Private by convention

Crossposter is not a public multi-tenant auth system. Keep it local, or protect it before exposing it over the internet.

  • Keep poster.config.local.json private.
  • Never commit API keys, OAuth tokens, browser cookies, passwords, or session files.
  • Use POSTER_REQUIRE_ADMIN_PASSWORD=true before public hosting.
  • Use accounts, boards, pages, channels, and profiles you own or are authorized to manage.

Privacy

What data stays local

Effective date: June 4, 2026

Crossposter stores configured credentials, titles, links, draft text, uploaded media, scheduled posts, and local publish history in the operator's own local or self-hosted environment.

Content is sent only to the social platforms selected by the operator. This website does not use analytics, advertising pixels, or tracking cookies.

Terms

Use it with accounts you control

Effective date: June 4, 2026

Use Crossposter only with accounts, boards, pages, channels, and profiles you own or are authorized to manage. You are responsible for your content and for each platform's rules, API terms, automation rules, and content policies.

Some integrations use official APIs. Others use local personal automation, browser cookies, or session files because the platform does not provide a normal public posting API for this use case.
